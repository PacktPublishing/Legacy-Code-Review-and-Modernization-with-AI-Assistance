# Review Security Risks

Review the selected code for security concerns.

Focus on:

- object-level authorization
- input validation
- error disclosure
- sensitive data exposure
- unsafe logging
- secrets or unsafe configuration
- dependency vulnerabilities
- insecure defaults
- cloud readiness concerns

Return evidence-based findings only. For each finding, include:

1. Location
2. Risk
3. Evidence
4. Potential impact
5. Remediation recommendation
6. Tests or tooling needed to validate the fix
