# Generate Characterization Tests

Generate tests that document the current behavior of the selected code before refactoring.

Requirements:

- Do not assume the behavior is correct.
- Capture current behavior first.
- Include edge cases and negative paths.
- Use clear test names.
- Prefer Arrange, Act, Assert structure.
- Identify any behavior that appears suspicious.

Return:

1. Test cases to add
2. Why each test matters
3. Suggested test code
4. Any unclear assumptions
